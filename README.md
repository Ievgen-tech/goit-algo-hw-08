# goit-algo-hw-08

## Домашнє завдання — Модуль 8: Дерева

### Завдання 1 — Пошук найменшого значення у BST

**Файл:** `modul1.py`

**Опис:**
Реалізація двійкового дерева пошуку (BST) з алгоритмом знаходження найменшого значення.

У BST мінімальний елемент завжди знаходиться у крайньому лівому вузлі, тому алгоритм просто рухається ліворуч від кореня до тих пір, доки не досягне вузла без лівого нащадка.

**Часова складність:** `O(h)`, де `h` — висота дерева.

---

### Запуск

```bash
python modul1.py
```

**Очікуваний вивід:**

```text
Значення у дереві: [15, 10, 20, 8, 12, 17, 25, 6]

Структура дерева:
        ┌── 25
    ┌── 20
    │   └── 17
┌── 15
│   │   ┌── 12
│   └── 10
│       └── 8
│           └── 6

Найменше значення в дереві: 6
```

---

### Додатково — Варіант для AVL-дерева

**Файл:** `modul12.py`

**Опис:**
Реалізація AVL-дерева (самобалансуючого BST) з тією ж функцією пошуку мінімального значення.

Мінімальний елемент так само шукається проходом до крайнього лівого вузла, але висота AVL-дерева підтримується збалансованою після вставок.

**Часова складність пошуку мінімуму:** `O(h)`, для AVL у середньому та найгіршому випадку `h = O(log n)`.

### Запуск AVL-варіанту

```bash
python modul12.py
```

**Очікуваний вивід:**

Структура AVL-дерева (збалансованого):
        ┌── 25
    ┌── 20
    │   └── 17
┌── 15
│   │   ┌── 12
│   └── 10
│       └── 8
│           └── 6

Найменше значення в дереві: 6

## Завдання 2 — Сума всіх значень у BST

**Файл:** `modul2.py`

**Опис:**
Реалізація двійкового дерева пошуку (BST) з алгоритмом обчислення суми всіх значень у дереві.

Алгоритм рекурсивно обходить усі вузли дерева і накопичує суму: `сума = поточний_вузол + сума_лівого_піддерева + сума_правого_піддерева`.

python modul2.py

    ┌── 20

┌── 15
│   │   ┌── 12
│   └── 10
│       └── 8
│           └── 6

## Завдання 3 — Мінімальні витрати на об'єднання кабелів

**Файл:** `modul3.py`

**Опис:**
Алгоритм знаходить порядок об'єднання кабелів, який мінімізує загальні витрати.

Використовується жадібна стратегія з мінімальною купою (min-heap):

1. Усі кабелі додаються у мінімальну купу.
2. На кожному кроці витягуються два найкоротших кабелі.
3. Витрати = сума їхніх довжин, результат повертається назад у купу.
4. Повторюється до тих пір, поки не залишиться один кабель.

**Часова складність:** `O(n log n)`, де `n` — кількість кабелів.

---

### Запуск (Завдання 3)

```bash
python modul3.py
```

**Очікуваний вивід:**

```text
Довжини кабелів: [4, 3, 2, 6]
Порядок з'єднань (від найменших до найбільших):
  Крок 1: 2 + 3 = 5
  Крок 2: 4 + 5 = 9
  Крок 3: 6 + 9 = 15

Мінімальні загальні витрати: 29
```

---

### Системні вимоги

- Python 3.10+
